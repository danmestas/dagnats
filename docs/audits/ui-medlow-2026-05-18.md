# ui-ux-pro-max audit — MEDIUM + LOW findings, 2026-05-18

Audit target: dagnats console at `http://127.0.0.1:8090/console/`, branch
`fix/console-ui-medium-low-bundle` based on `main` after PR #263
(0bc8919) — the CRITICAL/HIGH bundle.

The prior REPORT.md for this audit pass was lost. This file enumerates
the surviving MEDIUM + LOW issues by re-auditing the post-PR-#263 state
against ui-ux-pro-max Quick Reference + Norman principles. Each item is
shipped in this same PR.

Method: rendered every console page via the live binary on 8090, read
the actual HTML, traced each issue to its source file. Screenshots
saved alongside under `/tmp/dagnats-ui-medlow-screens/`.

Severity ladder: MEDIUM = visible inconsistency or bug affecting
trust/comprehension. LOW = polish item that does not block a task.

## MEDIUM

- **M1: Audit-log relative time wraps "minutes" past 60 minutes**
  (`internal/console/pages.go:839 formatDuration`). Renders
  `2092m27s ago` for a 35-hour-old entry. Cap should switch to hours,
  then days, then absolute date. Affects:
  `/console/ops/audit`, also any `formatDuration(time.Since(...))`
  caller. Fix: extend `formatDuration` to bucket into h, d, then date.

- **M2: Dashboard "Recent failures" + "Recent operator actions" panels
  show raw RFC3339 timestamps** (`templates/components/recent_panels.html:14,47`)
  instead of a humanised relative string. Inconsistent with the audit
  page (which is humanised, even if buggy). Fix: server-side
  `RelativeWhen` field that uses the same humaniser as M1.

- **M3: Detail-page back-link treatment is inconsistent across surfaces.**
  Ops pages have `<a class="page-back" href="/console/ops">← Ops</a>`
  at the top of the section header. Workflow / run / trigger / DLQ
  detail pages put the back link at the bottom in a bare `<p>`
  (`templates/workflow_detail.html:8,9`,
   `run_detail.html:9`,
   `trigger_detail.html:8,100`,
   `dlq_detail.html:69`).
  Operators have to scroll to navigate up. Fix: top-of-page
  `page-back` style on all detail pages.

- **M4: Inline `style="…"` violations on production templates.** Eight
  instances across `dlq_detail.html`, `trigger_detail.html`,
  `runs_list.html`, `components/dlq_action_modal.html`. Locks operators
  out of overriding via theme, defeats the e-ink editorial CSS scale,
  and creates drift between similar surfaces. Fix: hoist into
  `app.css` with named classes.

- **M5: 404 page (`/console/garbage`) loads layout chrome including the
  page-chrome connection pill stuck in "idle"** — fine for
  steady-state but means even on a missing page the operator sees the
  same nav with "DLQ" tooltip. Per the audit screenshots
  (`15-404-1440-LIGHT.png`), the 404 body is short; could surface the
  attempted path back as `<code>/console/garbage</code>` to
  confirm what was hit. Fix: include `Path` in 404 view.

- **M6: Connection pill stays "idle" on detail pages with no SSE
  binding.** Visually identical to "disconnected" if you don't
  remember the distinction. Fix: add a third visual treatment for
  "not applicable to this page" so it doesn't look like a broken
  connection — e.g. soften to a non-pill plain glyph, or hide on
  pages without SSE.

- **M7: Run-id truncation strategy is inconsistent.** Runs list table
  shows 12-char short (`6180318ffe04`), but dashboard "Recent
  operator actions" target column shows the full `trig-c0c5f72159b6d975`,
  and audit page "Target" shows full values too. Fix: introduce
  consistent shortening for IDs over 16 chars in dashboard recent
  actions + audit-log target column.

- **M8: Empty state for runs list when filters yield no rows
  is plain text only** ("no runs match these filters"
  — `fragments/runs_tbody.html:17`). Should suggest a fix
  (clear filters link). Fix: add a "clear filters" anchor in the
  empty row when filters are non-default.

## LOW

- **L1: `dashboard-recent-runid` link title attribute duplicates the
  inner short-id** — but the user can't see the long id without
  hovering. Replace with the short id rendered inline + the title for
  copy. (Same root as M7.)

- **L2: Sparkline column placeholder is a literal `·` `<span class="muted">`
  for every row.** Honest dead state but ungraceful. Fix: replace with
  a tiny visual hint that signals "no activity data" without ASCII
  noise (e.g. low-opacity dashed baseline).

- **L3: Cmd+K palette opens with an empty `<ul>` and no instructional
  copy** until the operator starts typing. Fix: a single line of
  hint copy before any input.

- **L4: Trigger toggle status pill swaps text+icon via inline JS
  manipulation of `innerHTML`** (`templates/trigger_detail.html:120-130`).
  Works but is fragile when icon font changes. Should be a server-rendered
  fragment swap via Datastar. Deferred — too deep for this bundle.
  Leaving here as known-tracked.

- **L5: Recent-failures item has `<time datetime="…">…</time>` where
  the visible text is the same ISO string as `datetime`.** Wastes the
  semantic value of `<time>`. Resolves with M2.

- **L6: DLQ detail "Error message" heading uses
  `<h3 style="margin-top:1rem;">`** — magic-number margin, also
  `font-size: 0.95rem`. Replace with a stylesheet rule, e.g.
  `.dlq-section-h3`. Resolves with M4.

- **L7: Trigger detail not-found state uses `<a class="btn">Return
  to triggers</a>` while DLQ detail not-found has no such link**
  (DLQ has no NotFound branch in template at all because the data
  source bounces back to 404). Minor copy/treatment inconsistency.
  Leave — out of scope.

- **L8: Audit-log empty filter copy lives in `audit_log.html:37-49`
  but uses imperative voice while runs-list empty copy
  (`runs_list.html:62-72`) uses different framing.** Same pattern
  unification would help. Out of scope for this bundle — too far
  into copy editing.
